1. books.append(new_book)는 평균 O(1)인데 add_book() 전체가 O(n)인 이유는 무엇인가?
append는 시간 복잡도가 평균 O(n)이지만, save_books를 이용하여 도서 전체를 저장하여 O(1) + O(n) = O(n)이므로 전체가 O(n)이다.
[수정 후]
append는 평균 O(1)이지만, add_book()에서 save_books()를 호출하여 도서 목록 전체를 JSON 파일에 다시 저장한다. save_books()가 O(n)이므로 O(1) + O(n) = O(n)이 되어 add_book() 전체도 O(n)이다.

2. 도서가 한 권만 추가됐는데도 save_books()가 전체 목록을 다시 기록하는 이유는 무엇인가?
w 모드는 파일의 안에 있는 내용의 길이를 0으로 만든 버린다. 그 후 json.dump(books, file, ...)을 통해 JSON 구조 전체를 다시 기록한다.

3. search_books()에서 첫 번째 도서가 검색되더라도 반복을 끝낼 수 없는 이유는 무엇인가?
일치하는 모든 도서를 찾기 위해서 파이썬 객체를 끝까지 순차 탐색하기 때문이다.

4. 현재 JSON 프로그램에서 도서 수가 매우 많아지면 추가·검색·실행 시작이 느려지는 이유는 무엇인가?
json.load()와 json.dump()를 하면 할 때 마다 전체 파일 읽기를 하고, 전체 파일 쓰기를 하기 때문에 그 때 마다 순차 검색으로 요소를 하나씩 확인 해줘야 하기 때문이다.
[수정 후]
프로그램을 시작할 때 json.load()가 JSON 파일 전체를 읽는다. 도서를 추가할 때는 save_books()가 도서 목록 전체를 파일에 다시 기록한다. 검색할 때는 모든 도서를 처음부터 끝까지 순차 탐색한다. 따라서 도서 수가 많아질수록 전체 파일 읽기, 전체 파일 쓰기, 순차 검색에 걸리는 시간이 증가한다.
