##수정 전 써둔 것
pwd: 내가 터미널에서 현재 무슨 폴더에 있는지 알려줌
ls: 지금 사용하고 있는 파일의 세부내용을 알려줌
cd: change dictionary 내가 있는 위치를 cd 뒤의 주소로 바꿔줌
touch: 폴더 안에 파일을 형성한다
git status: git에 있는 자료의 상태를 확인한다.
git add: 깃에 자료를 추가 할 것이라고 알려주는 용도
git commit: 깃에 자료를 추가한다.

##수정 후 써둔 것
pwd: 내가 터미널에서 현재 무슨 폴더에 있는지 알려줌
ls: 현재 폴더 안에 있는 파일과 폴더의 목록을 보여줌
파일의 세부 내용을 보여주는 것이 아니라 "폴더 안에 있는" "파일과 폴더의 목록"을 보여준다.
cd: change directory: 현재 위치를 다른 폴더 위치로 이동한다.
touch: 파일이 없으면 빈 파일을 만든다. 있으면 시간을 수정한다.
git add: 파일의 현재 변경 내용을 다음 commit에 포함할 대상으로 올린다.
다음 기록에 넣을 '변경 내용'을 '고른다'
Git은 실제로는 현재 파일의 변경된 상태를 스테이징 영역에 올리는 것에 가깝기 때문이라는데 이것은 그냥 적어만 두자
git commit: git add로 올린 변경 내용을 하나의 기록으로 저장한다.
'고른 변경 내용'을 '실제 기록'으로 남긴다.
