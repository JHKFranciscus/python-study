## 만든 프로그램

점수를 저장하고 관리하는 프로그램을 생성

프로그램 추가기능:
점수 추가
전체 점수 보기
점수 개수, 총합, 평균, 최고점, 최저점 보기
60점 이상 합격자 수 보기

점수는 리스트에 저장했고, `while` 반복문을 사용해 사용자가 종료를 선택할 때까지 메뉴가 계속 나오도록 만들었다.

주말에는 이 프로그램을 원본 코드 없이 다시 작성

---

## 가장 많이 막힌 부분

새로 입력받은 점수를 리스트에 추가하는 부분

첫 작성:

```python
scores = new_score
```

의도: 새 점수를 `scores` 리스트에 추가하는 것
실제:
`scores`라는 변수가 리스트가 아니라 정수를 가리키도록 바뀜

그 상태에서 전체 보기 메뉴를 실행하자 다음 오류가 발생

```text
TypeError: object of type 'int' has no len()
```

오류를 확인했지만, 처음에는 왜 `scores`가 정수가 되었는지 바로 찾지 못했다.

---

## 문제의 원인

설명을 받은 뒤:
`scores = new_score`는
리스트에 값을 추가하는 코드가 아니라, 
`scores` 자체를 `new_score` 값으로 바꾸는 코드

수정하는 과정에서도 다시 다음처럼 잘못 작성

```python
scores.append(scores)
```

이 코드는 새 점수가 아니라 리스트 자기 자신을 리스트 안에 넣는 코드

실행 결과가 다음과 비슷하게 나왔다.

```text
[[...], [...]]
```

이상한 출력은 직접 확인했지만, 왜 이런 형태가 나오는지는 설명을 받은 뒤 이해함.

새로 입력받은 점수를 넣으려면 `scores`가 아니라 `new_score`를 `append` 안에 넣어야 함.

## 내가 직접 확인한 과정

1.반복문 밖에서 빈 점수 리스트를 만들었다.
2.`while` 반복문으로 메뉴를 계속 출력했다.
3.점수 추가 메뉴에서 80점과 95점을 입력했다.
4.전체 보기 메뉴를 실행했다.
5.`TypeError` 오류와 이상한 리스트 출력을 확인했다.
6.설명을 받은 뒤 코드를 수정했다.
7.프로그램을 다시 실행했다.
8.전체 보기에서 `[80, 95]`가 출력되는 것을 확인했다.
9.50점, 80점, 100점을 넣고 통계 기능도 실행했다.
10.마지막 점수 삭제 기능을 실행해 100, 80, 50 순서로 삭제되는 것을 확인했다.

오류 원인을 혼자 분석해 찾은 것은 아니지만,
수정 전과 수정 후의 실행 결과는 직접 비교했다.

---

## 수정한 방법

잘못 작성한 코드는 다음 두 가지

```python
scores = new_score
```

```python
scores.append(scores)
```

최종 수정본:

```python
scores.append(new_score)
```

`new_score`는 사용자가 새로 입력한 점수
`scores`는 여러 점수를 저장하는 리스트

통계 기능에서도 변수를 초기화하는 위치를 다시 확인

`total`, `highest_score`, `smallest_score`를 반복문 안에 뒀는데,
반복할 때마다 값이 다시 처음 상태로 돌아간다.

다음처럼 반복문 전에 초기화

```python
total = 0
highest_score = scores[0]
smallest_score = scores[0]

for score in scores:
    total += score
```

누적하거나 이전 값과 비교해야 하는 변수는
반복문 전에 만들어야 한다는 것을 재확인

## 실제 실행 결과

점수로 50, 80, 100을 입력한 뒤 통계 기능을 실행한 결과는 다음과 같았다.

```text
현재 점수: [50, 80, 100]
점수 개수: 3
총합: 230
평균: 76.66666666666667
최고점: 100
최저점: 50
합격자 수: 2
```

마지막 점수 삭제 기능을 실행하자 가장 나중에 넣은 100점이 먼저 삭제됨.

이후 다시 삭제했을 때 80점, 50점 순서로 삭제됨
리스트가 비어 있는 상태에서는 프로그램이 오류로 종료되지 않고 다음 안내가 나옴

```text
삭제할 점수가 없습니다.
```

점수가 없는 상태에서 전체 보기와 통계 메뉴를 실행했을 때도 별도의 안내 문장이 출력되도록 처리함
