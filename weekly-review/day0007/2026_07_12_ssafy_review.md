# 문제 1 — 백분율

어떤 상품의 원래 가격은 80,000원이다.
가격을 15% 인상했다.

인상된 금액은 얼마인가?
인상 후 가격은 얼마인가?

- 답: 인상된 금액: 12,000원, 인상 수 가격: 92,000원
- 계산 과정: 80,000 * 0.15 = 12,000  80,000 + 12,000 = 92,000
- 증가한 금액과 최종 가격을 어떻게 구분했는가: 원래 가격에서 인상한 가격만 하면 증가한 금액이고, 최종가격은 원래 가격에 증가한 금액을 더한 것이다. 

# 문제 2 — 작업량

A는 어떤 일을 혼자 하면 6시간이 걸리고,
B는 같은 일을 혼자 하면 3시간이 걸린다.

두 사람이 함께 일하면 몇 시간이 걸리는가?

- 답: 2시간
- A가 1시간 동안 하는 일의 양: 1/6
- B가 1시간 동안 하는 일의 양: 1/3
- 둘이 1시간 동안 하는 일의 양: 1/2
- 계산 과정: 1 / (1 / 6 + 1 / 3) = 2

# 문제 4 — 명제·조건 추론

다음 조건이 참이다.

프로그래밍 문제를 모두 풀면 복습을 한다.

각 문장을 반드시 참, 반드시 거짓, 알 수 없음 중 하나로 판단하시오.

1. 민수가 프로그래밍 문제를 모두 풀었다. 그러므로 민수는 복습을 한다.
2. 민수가 복습을 했다. 그러므로 민수는 프로그래밍 문제를 모두 풀었다.
3. 민수가 프로그래밍 문제를 모두 풀지 않았다. 그러므로 민수는 복습을 하지 않았다.
4. 민수가 복습을 하지 않았다. 그러므로 민수는 프로그래밍 문제를 모두 풀지 않았다.

- 1번 판단과 이유: 참, 조건과 일치하기 때문이다
- 2번 판단과 이유: 알 수 없다. 조건의 역인데 조건이 참일 때 그 조건의 역은 참인지 거짓인지 알 수 없다.
- 3번 판단과 이유: 알 수 없다. 조건의 이인데 조건이 참일 때 그 조건의 이는 참인지 거짓인지 알 수 없다.
- 4번 판단과 이유: 참, 조건의 대우인데 조건이 참일 대 그 조건의 대우는 참이다.

문제 5 — 8비트 정수 범위

다음 질문에 답하시오.

1. 8비트 unsigned 정수의 범위
2. 8비트 signed 정수의 범위
3. signed 8비트에서 음수는 몇 개인가?
4. 0은 몇 개인가?
5. 양수는 몇 개인가?
6. 최솟값은 -128인데 최댓값은 127인 이유

- 1번: 0 ~ 255
- 2번: -128 ~ 127
- 3번: 128개
- 4번: 1개
- 5번: 127개
- 6번 설명: signed 8비트의 최상위 비트는 -128의 자리값을 가진다. 그 뒤는 차례로 64, 32, 16, 8, 4, 2, 1의 자리값을 가지기 때문이다.

- 2의 보수 signed 8비트에서 최상위 비트는 -128의 자릿값을 가진다.
- 최솟값은 최상위 비트만 1인 `10000000`이므로 -128이다.
- 최댓값은 최상위 비트가 0이고 나머지 7비트가 모두 1인
  `01111111`이므로 64 + 32 + 16 + 8 + 4 + 2 + 1 = 127이다.
- 전체 256개 중 음수 128개, 0 한 개, 양수 127개이기 때문에
  최솟값과 최댓값의 절댓값이 같지 않다.

  ## 재풀이 결과

- 5문제 모두 정답이었다.
- 문제 1은 계산은 맞았지만 두 질문의 답을 답란에 분리해 쓰지 않았다.
- 문제 2는 계산 결과를 구했지만 답란을 비워 두었다.
- 문제 5는 범위와 개수는 기억했지만 최댓값 127이 만들어지는 비트 구조 설명이 부족했다.
- 다음에는 계산 과정뿐 아니라 문제에서 요구한 답을 빠짐없이 별도로 적는다.
