# 2단계. 대표 문제 하나를 확정하기 — 3분

이번 글의 중심 문제는 이것으로 고정합니다.

점수 관리 프로그램을 독립 재현하면서 scores, new_score, append의 역할을 혼동한 문제

다른 Git·SSAFY·CS 내용은 “이번 주 배운 내용”에만 짧게 넣고, 본문에서는 이 문제를 중심으로 씁니다.

# 3단계. 직접 한 일과 도움받은 일을 분리하기 — 5분

내가 직접 한 것
빈 화면에서 프로그램 구조 작성
프로그램 실행
오류 메시지 확인
[[...], [...]] 출력 확인
수정된 코드 입력
[80, 95] 정상 출력 확인
통계와 삭제 기능 실행
설명을 받아 알게 된 것
scores = new_score가 리스트를 정수로 바꾼다는 원인
scores.append(scores)가 자기 자신을 리스트에 넣는다는 원인
scores.append(new_score)가 올바른 코드라는 점
누적·비교 변수는 반복문 전에 초기화해야 한다는 점

다음 표현은 쓰지 않습니다.

오류를 분석해서 원인을 직접 찾아 해결했다.

사실에 맞는 표현:

오류와 이상한 출력은 직접 확인했지만 원인은 바로 찾지 못했다.
설명을 받은 뒤 원인을 확인하고 코드를 수정했으며,
다시 실행해 정상 결과가 나오는 것을 확인했다.

# 4단계. 고정 형식 9개 항목 채우기 — 15분

파일을 위에서부터 순서대로 작성합니다.

1. 이번 주 배운 내용
Python·Git·CS·SSAFY를 각각 1~2문장으로 정리합니다.

2. 만든 프로그램
프로그램의 목적과 기능을 설명합니다.

3. 가장 많이 막힌 부분
append를 이용한 점수 저장 문제 하나만 구체적으로 적습니다.

4. 문제의 원인
직접 알아낸 내용과 설명을 받아 알게 된 내용을 구분합니다.

5. 내가 직접 확인한 과정
입력 → 오류 발생 → 수정 → 재실행 순서로 적습니다.

6. 수정한 방법
수정 전과 수정 후 코드를 포함합니다.

예:
잘못 작성
scores = new_score
scores.append(scores)

수정
scores.append(new_score)
7. 실제 실행 결과

다음을 포함합니다.

현재 점수: [50, 80, 100]
점수 개수: 3
총합: 230
최고점: 100
최저점: 50
합격자 수: 2
8. 아직 부족한 부분

현재 혼자 하지 못한 부분을 사실대로 적습니다.

9. 다음 주 목표

측정 가능한 행동 2~3개만 정합니다.
