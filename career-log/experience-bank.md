## 2026-07-08 조건식 오류 수정

### 상황
시험 점수와 출석률을 각각 정수로 입력받고
int(), input(), if ,elif ,else ,and ,or을 사용하여
점수 또는 출석률이 0 미만이거나 100 초과이면
잘못된 입력입니다.
점수가 70 이상이고 출석률이 80 이상이면
수료입니다.
그 외에는
미수료입니다.
위 조건을 만족하는 수료 판정 프로그램을 작성했다.

### 문제
점수 105, 출석률 90을 입력했을 때
잘못된 입력이 아니라 수료라고 출력됐다.

### 내가 처음 시도한 방법
두 변수의 범위를 한꺼번에 검사하려고 (score or check_rate) < 0 and (score or check_rate) > 100을 사용했다.

### 잘못된 이유
`score or check_rate`는 두 변수를 모두 검사하는 표현이 아니었다.
두 값 중 참으로 취급되는 한 값을 반환하므로, 각 변수의 범위를 따로 비교하지 못했다.

또한 하나의 값이 동시에 0보다 작고 100보다 클 수 없으므로
`값 < 0 and 값 > 100` 조건은 항상 거짓이었다.

### 수정한 방법
score와 check_rate를 각각 비교하도록 조건식을 score < 0 or score > 100 or check_rate < 0 or check_rate > 100으로 수정함.
점수 85·출석률 105, 점수 85·출석률 -1을 입력해 다시 확인함.

### 결과
점수 85·출석률 105와 점수 85·출석률 -1을 입력했을 때
모두 `잘못된 입력입니다.`가 출력되는 것을 확인했다.
정상 범위의 입력에서는 수료와 미수료도 기준에 맞게 출력됐다.

### 이 경험에서 배운 점
여러 변수의 범위를 검사할 때는 변수마다 비교식을 따로 작성해야 한다.
코드가 실행된다는 것만으로 맞다고 판단하지 말고,
105, -1 같은 잘못된 입력과 경계값을 직접 넣어 확인해야 한다.



## 2026-07-09 Day 4 - 조건문을 배우면서 헷갈린 개념을 정리한 경험

### 상황
Python 조건문을 배우면서 if / elif / else, input, int, and 조건을 연습했다.  
처음에는 코드를 바꿔서 테스트해야 하는데, 기존 코드 아래에 계속 추가해서 실행 결과가 여러 번 나오는 실수를 했다.

### 문제
조건문 자체보다도 코드를 어떻게 테스트해야 하는지, 그리고 =와 ==의 차이를 구분하는 것이 헷갈렸다.  
또 input으로 받은 값이 바로 숫자가 아니라 문자열이라는 점도 처음에는 익숙하지 않았다.
if ticket == yes (x) if ticket == "yes" (o)

### 내가 한 행동
코드를 한 번에 계속 추가하지 않고, 기존 값을 바꿔가며 다시 실행하는 방식으로 고쳤다.  
점수 등급, 나이 판별, 짝수/홀수 판별, 티켓 입장 조건 문제를 직접 실행해 보면서 조건문 구조를 반복했다.  
또 공부일지에 =는 값 저장, ==는 비교, signed integer와 unsigned integer의 차이를 따로 정리했다.
비교 연산자에서 정의된 변수가 아닌 문자열에는 " "를 붙여주었다.

### 배운 점
조건문은 단순히 문법을 외우는 것이 아니라, 문제의 조건을 코드의 비교식으로 바꾸는 연습이 필요하다는 것을 알게 되었다.  
또 input은 문자열로 들어오기 때문에 숫자 비교를 하려면 int로 바꿔야 한다는 점을 알게 되었다.

### 자소서나 면접에 쓸 수 있는 포인트
기초 개념을 배울 때 실수를 그냥 넘기지 않고, 왜 틀렸는지 기록하고 다시 실행하면서 수정했다.  
처음에는 따라 치는 수준이었지만, 점점 문제 조건을 코드로 바꾸는 방식에 익숙해지고 있다.

## 2026-07-11 - 통계 변수의 초기화 위치를 수정한 경험

### 1. 상황

리스트를 이용한 점수 관리 프로그램을 만들면서
저장된 점수의 합계, 평균, 최댓값, 최솟값을 출력하는 기능을 구현하고 있었다.

### 2. 문제

처음에는 total을 scores처럼 while문 밖에 두어도 된다고 생각했다.
하지만 total을 프로그램 시작 때 한 번만 0으로 초기화하면,
통계 메뉴를 반복 실행할 때 이전 합계에 같은 점수가 다시 더해질 수 있었다.

### 3. 행동

scores와 total이 각각 언제까지 값을 유지해야 하는지 비교했다.
scores는 입력한 점수를 계속 저장해야 하므로 while문 밖에 두고,
total은 통계를 실행할 때마다 다시 계산하도록 3번 메뉴 안에서 0으로 초기화했다.
이후 같은 점수에서 통계 메뉴를 반복 실행해 결과를 확인했다.

### 4. 결과

점수 [80, 50, 100]에서 통계 메뉴를 실행했을 때
합계 230, 평균 76.66666666666667, 최댓값 100, 최솟값 50이 출력됐다.
통계를 다시 실행해도 합계가 중복되지 않고 같은 결과가 나왔다.

### 5. 배운 점

변수의 위치는 단순히 반복문 안과 밖으로 외우는 것이 아니라,
값을 얼마나 오래 유지해야 하며 언제 다시 초기화해야 하는지를 기준으로 정해야 한다.
앞으로 변수를 만들 때 프로그램 전체에서 유지할 값인지,
특정 기능을 실행할 때마다 초기화할 값인지 먼저 판단하겠다.
