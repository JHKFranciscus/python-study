# 파일과 메모리

## 1. 변수와 파일의 차이
변수는 실행 중이 프로그램의 데이터를 담지만,
파일에 저장하는 것은 프로그램이 종료되어도 데이터가 남는다.

## 2. 파일을 읽을 때 데이터가 이동하는 과정
1.파일을 연다.
2.with 블록이 시작되면 open이 파일에 저장된 객체를 하나의 문자열로 만들어서 메모리에 새로운 독립된 문자열 객체를 생성한다.
3.변수를 사용하며 문자열을 읽으면 메모리에 저장되어 있는 문자열을 읽는다.
[수정 후]
open()
→ 파일을 사용할 수 있는 파일 객체를 만든다.
file.read()
→ 파일 내용을 읽어 하나의 문자열을 반환한다.

## 3. read()와 줄 단위 읽기의 차이
read()는 파일에 저장된 객체를 하나의 거대한 문자열로 만들어서 읽지만, 줄 단위 읽기는 파일에 저장 된 객체를 하나의 거대한 문자열로 만들어서 메모리에 저장하는 것까지는 같은데 메모리에 저장된 객체를 줄 별로 나누어 읽는다.
[수정 후]
read()
→ 파일 전체를 하나의 문자열로 읽는다.
for line in file
→ 파일에서 줄을 차례대로 가져와 처리한다.
→ 전체 파일을 하나의 문자열로 먼저 저장하지 않는다.

## 4. 시간 복잡도와 메모리 사용량의 차이
시간 복잡도와 메모리 사용량은 메모리 사용량이 늘어나도 시간복잡도와는 관계가 없다.
[수정 후]
둘은 서로 다른 평가기준이다.
시간 복잡도
→ 입력 크기가 증가할 때 실행 작업량이 어떻게 증가하는지 나타낸다.
메모리 사용량
→ 실행 중 필요한 저장 공간이 얼마나 증가하는지 나타낸다.

## 5. with open()을 사용하는 이유
.write()메소드를 사용한다고 바로 파일에 값이 저장되는 것이 아니라 버퍼에 임시로 저장되었다가 with블록이 끝나면, 즉 파일이 닫히면 그 때 파일에 저장하기 때문이다.


# 확인 문제

1. content = file.read()에서 content는 파일 자체를 가리키는가?
아니다. 메모리에 복사된 파일의 내용을 하나의 거대한 새롭고 독립적인 문자열 객체로 만든 객체를 가리킨다.
[수정 후]
file.read()가 파일에서 읽어 반환하는 문자열 객체를 가리킨다.

2. 프로그램을 종료하면 memo 변수는 사라지는데 memos.txt의 내용은 남는 이유는 무엇인가?
프로그램을 종료하면 메모리에 남아있는 데이터와 명령어는 사라지지만 외부저장장치인 파일에 존재하는 내용은 사라지지 않기 때문이다.

3. 10GB 파일을 read()로 한 번에 읽는 방식이 위험할 수 있는 이유는 무엇인가?
자료의 줄이 구분이 제대로 되지 않아 사람이 보기에 가독성이 떨어져서
[수정 후]
핵심 문제는 메모리 부족이다.
10GB 파일 전체를 하나의 문자열로 만들려면 메모리에 매우 큰 공간이 필요하다.
사용 가능한 메모리가 부족하면 프로그램이 느려지거나 MemoryError가 발생하거나 운영체제가 프로그램을 종료할 수 있다.

4. 파일을 한 줄씩 읽더라도 모든 줄을 검색하면 시간 복잡도를 O(n)이라고 보는 이유는 무엇인가?
시간 복잡도는 데이터양과 상관없이 한 줄 씩 반복으로 읽기 때문이다.
[수정 후]
입력 데이터 개수와 작업량이 비례한다.
파일에 n개의 줄이 있고 모든 줄을 확인하면 줄 수가 늘어나는 만큼 반복 횟수도 늘어난다.
따라서 시간 복잡도를 O(n)이라고 본다.

5. read() 방식과 줄 단위 방식이 모두 O(n)일 수 있는데도 줄 단위 방식이 메모리 면에서 유리할 수 있는 이유는 무엇인가?
한 줄 씩 읽기 때문이다.
[수정 후]
줄 단위 방식은 파일 전체를 하나의 문자열로 보관하지 않고 줄을 차례대로 처리하므로 메모리 사용량이 더 작을 수 있다.

6. file.write() 뒤에 with 블록이 종료되면 어떤 처리가 이루어지는가?
메모리의 버퍼에 저장되어 있던 값들이 파일에 저장이된다.
[수정 후]
with 블록이 종료되면 남아 있는 버퍼의 내용이 전달되고 파일이 자동으로 닫힌다.
