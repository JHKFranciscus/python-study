## 막힌 점
find_book()이 반환한 딕셔너리가 복사본인지 원본인지 설명할 때 처음에는 복사한다고 잘못 생각했다.
copy.()에서 복사하는 부분을 헷갈렸다.
순차 탐색이 O(n)인 이유를 데이터 크기와 비교횟수를 연관지어 정확하게 설명하지 못했다.
숫자를 비교하는데 ==이 아닌 is를 사용하여 SyntaxWarning가 발생했다.

## 내가 시도한 것
원본 딕셔너리를 반환하는 함수와 copy()를 반환하는 함수를 각각 실행해 수정 결과를 비교했다.
O(n)의 정확한 의미를 파악하기 위해 비교 횟수와 데이터 크기의 관계를 다시 확인했다.
is로 실행하여 SyntaxWarning이 발생하여 ==로 수정해 다시 실행했다.

## 알게 된 것
return book은 원본 dictionary 객체를 반환하므로 반환값을 넣은 변수를 통해서 수정해도 books 안의 원본도 바뀐다.
없는 key를 []로 조회하면 KeyError가 발생하지만 get()은 None이나 지정한 기본값을 반환한다.
dictionary를 그대로 반복하면 key가 나오고 items()를 사용하면 key와 value를 함께 받을 수 있다.
O(n)은 입력 크기 n이 커질 때 필요한 연산 횟수가 얼마나 증가하는지를 나타낸다. 순차 탐색은 최악의 경우 n개의 데이터를 모두 비교하므로 시간복잡도가 O(n)이다
값이 같은지 비교할 때는 ==를 사용하고 is는 같은 객체인지를 확인할 때 사용한다.
