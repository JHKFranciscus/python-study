## 막힌 부분

- open()과 read()의 역할을 처음에 정확히 구분하지 못했다.
- 독립 재현에서 공백 입력 처리와 출력 줄바꿈 처리를 소문자로 변경처리 메소드와 함께 썼다.
- 빈 검색어를 처리한 뒤 return을 사용하지 않아 다른 안내도 함께 출력됐다.
- for memo_number, file in enumerate(file, start=1): 이라고 file을 중복해서 썼다.

## 시도한 것

- open()은 파일 객체를 반환하고 read()가 문자열을 반환한다는 내용으로 수정했다.
- 공백 입력 처리와 출력 줄바꿈 처리를 한 변수와 그 변수에 다시 소문자 변경처리 메소드를 한 변수를 만들었다.
- 빈 검색어 처리 뒤 return으로 함수를 종료했다.
- 앞의 지역변수 file을 line으로 고쳐서 중복의 오류를 수정했다.

## 배운 것

- w는 기존 파일을 열 때 기존 내용을 지우고 파일 크기를 0으로 만든다. a는 무조건 파일 끝에 내용을 추가한다.
- 프로그램이 종료되면 변수는 유지되지 않지만 파일 내용은 저장장치에 남는다.
- read()는 전체 파일을 문자열로 읽고, 파일 객체를 순회하면 줄 단위로 처리할 수 있다.
- 두 방식 모두 전체 줄을 검색하면 O(n)일 수 있지만 메모리 사용 방식은 다르다.
